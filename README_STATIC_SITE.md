# Static site (GitHub Pages) — сборка и деплой

Что добавлено:
- Workflow GitHub Actions: собирает проект при пуше в ветку `main` и публикует `dist/` в ветку `gh-pages`.

Как это работает:
1. Закоммитите и запушьте изменения в ветку `main`.
2. GitHub Actions запустит сборку и задеплоит папку `dist` в ветку `gh-pages`.
3. Через пару минут сайт будет доступен по адресу:
   https://Rusmer.github.io/regruha/

Локальная проверка:
- Установите зависимости: `npm ci`
- Соберите проект: `npm run build`
- Проверить содержимое `dist/` локально (например, `npx serve dist`)

Примечания:
- Если вы хотите использовать кастомный домен, добавьте `CNAME` в корень `dist/` или настройте через репозиторий GitHub Pages.
- Если сборка требует специальных переменных окружения/Base44 backend, workflow может падать — пришлите лог Actions, я помогу исправить.
